1 我们做什么

一个装进酒店客房的硬件终端 + 云端平台，让酒店「感知」房间状态，自动做节能与安防。

⚡ 节能

人走自动关空调/关灯，入住自动开。1间房1年省¥800~1,500电费。

50间房的小酒店，一年白省¥4~7.5万。

🛡️ 安防

雷达感知房间里「有几个人」「待了多久」。异常静默、陌生人闯入、衣帽间有人停留过久——实时告警。

酒店老板手机收到推送，看一眼就知道。

根能力是「空间感知」—— 随时知道房间里有没有人、有几个人、在做什么区域。节能和安防是感知之上的两个应用。

2 产品形态

一个巴掌大的终端，装在天花板上

硬件组成：

  • ESP32-S3 主控（双核240MHz / 16MB Flash）

  • 无人15分钟→自动关空调（可配置）

4 市场痛点

🏨 酒店的痛

  • 客人退房不关空调，每天约1~3间房浪费2~4小时，一间损失约¥3~5
  • 50间房一年因空调浪费损失约¥2,000~5,000
  • 客房安防靠人工巡检，死角多、成本高
  • 员工私用房、衣帽间被盗、保洁时间无法追溯

✅ 灵犀的解法

  • 无人感知→自动关空调/灯光 → 省电30~50%
  • 人去哪了→看得见房间状态 → 减少浪费
  • 异常有人→实时推送手机 → 老板放心
  • 一个人（序章+AI）就能管全部开发和部署

5 技术栈

ESP32-S3 ESP-IDF LD2450 雷达 ML307C 4G MQTT UART OTA Python FastAPI WebSocket MySQL GitHub Pages

全栈：端侧 C++ (ESP-IDF) → 通信层 MQTT+AT → 后端 Python/FastAPI → 管理页面 HTML/JS → 展示页 GitHub Pages

终端侧（ESP32-S3）

  • 雷达驱动（UART）→ 坐标/人数/微动/轨迹
  • 红外库（格力/美的/海尔等）→ 空调控制
  • 4G模组（ML307C）→ MQTT上报+接收指令
  • OTA升级 + 安全启动（规划中）
  • NVS可配（MQTT服务器/策略等）

云端

  • MQTT Broker（EMQX）→ 设备消息队列
  • FastAPI → 设备管理/传感器入库/PMS对比
  • MySQL → 设备表/传感器流/节能报表
  • 管理后台 → 房间管控/策略配置/省电看板
  • 员工端 → 保洁签到/房间状态查看
